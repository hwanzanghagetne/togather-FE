## 2026-06-27 | 배포 환경별 프록시 분리 경험

**상황**
로컬에서는 로그인과 API 호출이 정상인데, Vercel 배포본에서만 카카오 로그인과 `/api` 요청이 막혔다. 같은 프론트 코드라도 개발 서버와 배포 서버가 요청을 우회시키는 방식이 다르다는 점이 원인이었다.

**고민**
문제가 프론트 버튼 로직인지, OAuth 백엔드 인증 처리인지, 아니면 배포 플랫폼의 rewrite 누락인지 분리해서 봐야 했다. 특히 로컬은 `vite.config.ts` proxy를 쓰지만 배포는 그 설정을 전혀 읽지 않는다는 점을 다시 확인했다.

**선택**
Vercel 배포 환경에서 실제로 사용되는 `vercel.json` rewrite를 Cloudtype 백엔드 주소로 명시했다. 추가로 현재 브랜드 선택안인 B 로고를 기준으로 PWA manifest 아이콘도 맞췄다.

**결과**
배포 이슈를 프론트 코드 로직 문제로 오해하지 않고, 개발 환경과 배포 환경의 요청 라우팅 차이로 설명할 수 있게 됐다. 이후에는 배포 장애가 나면 어느 설정 파일을 봐야 하는지 빠르게 좁힐 수 있다.
