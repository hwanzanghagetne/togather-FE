## 2026-06-27 | 배포 로그인 경로 점검
- AI 제안: 로컬은 `vite.config.ts`, 배포는 `vercel.json`이 각각 프록시와 리라이트를 담당하므로 배포 로그인 문제는 `vercel.json`의 실제 백엔드 연결부터 확인하자고 제안.
- 내가 수정: 사용 중인 Cloudtype 백엔드 주소를 rewrite 목적지에 반영하고, 현재 쓰는 B 로고 기준으로 PWA 아이콘 파일도 정리했다.
- 판단 근거: 로그인 버튼 코드는 상대 경로(`/oauth2/...`)를 사용하고 있어 배포 환경에서는 Vercel rewrite가 정확해야 OAuth 서버로 정상 이동한다.

## 2026-07-02 | 입력 영역 레이아웃 재조정 및 iOS 자동 확대 전체 대응
- AI 제안: `fontSize: 16px` 적용이 채팅에만 되어 있고 ProfileEdit·CreateMeetup·Step4Profile에는 14~15px가 남아 있어 동일 확대 현상이 발생할 수 있으니 한꺼번에 수정하자고 제안. 동시에 ChatRoomPage footer 버튼 터치 타겟(24→40px)과 sendButton(40→44px)도 Apple HIG 최소 기준에 맞게 조정.
- 내가 수정: 특별한 수정 없이 제안 그대로 진행.
- 판단 근거: iOS 자동 확대의 주 원인이 입력 필드 font-size < 16px이므로, 채팅 외 입력 화면에도 같은 원칙을 일괄 적용하는 것이 효율적이라 판단.

## 2026-07-02 | iPhone 채팅 입력 확대 이슈 정리
- AI 제안: iPhone Safari/PWA에서 입력 포커스 시 자동 확대는 보통 `16px` 미만 입력 폰트에서 발생하므로 채팅 입력창만 먼저 `16px` 기준으로 올려서 확인하자고 제안.
- 내가 수정: 채팅 입력창 폰트와 높이, 전송 버튼 크기를 조정해 배포 반영 여부를 먼저 확인하고, 전체 프로젝트 전역 확대 차단 대신 채팅 화면부터 단계적으로 검증하기로 결정했다.
- 판단 근거: 배포 화면에서 하단 채팅 바 디자인이 이미 바뀐 것이 확인되어 최신 배포는 반영된 상태였고, 확대가 계속된다는 점에서 캐시보다 iOS 입력 영역 동작과 레이아웃 구조 쪽을 우선 의심하는 것이 맞다고 판단했다.
